JavaScript: E4X

E4X, or ECMAScript for XML is an extension of JavaScript that makes XML part of JavaScript syntax and provides a number of mechanisms for traversing and modifying XML data.

XML in Alt

The Alt Framework makes heavy use of XML for both SQueaL's SQL Schema and the Onion ML markup system. E4X's native JavaScript approach became the logical approach for working with XML in the Alt Framework.

E4X Documentation

Finding useful resources or references on E4X is not easy, many simply pointing to the original highly technical ECMA specification.

The best documentation I have found is the Adobe documentation on ActionScript (also based on ECMAScript) which has just recently been given E4X support.

This most likely reason for a lack of documentation is the lack of usefulness. Only Firefox 1.5+ supports E4X natively in JavaScript, and the complexity of E4X makes it difficult to 'wrap' for other browsers.

Quick Reference

This reference is not intended to be official, it is simply my interpretation of various features in E4X.

XML object

Constructed by either writing literal XML code in your script: <foo>bar</foo>, or with the string-based constructor: new String("<foo>bar</foo>").

The literal XML syntax allows you to refer to JavaScript variables in XML blocks by using curly braces. E.g. <foo>{somevar}</foo>. Note that for tag attributes you need to omit the quotes (e.g. <foo id={somevar}/>). Scripts can be specified for tag and attribute names as well (e.g. <{tagName} {attributeName}={attributeValue}/> is valid).

You can retrieve attributes from an XML object by using the syntax xml.@name, where xml is the XML object and name is the name of the attribute.

You can retrieve a child XML tag (also a XML object) with the syntax xml.node, where xml is the XML object, and node is the name of the child tag. If there are multiple children with the same tag, this exact same syntax instead returns a XMLList object.

XMLList object

This object acts like an array with one main difference: length is a method and not a property. That means to get the number of XML nodes in a XMLList object, you need to use xmllist.length().

E4X Websites

Recommended Resources

These are the links I recommend reading to get a feel for E4X from several sources.

AJAX and scripting Web services with E4X - A tutorial from IBM's developerworks site. A good starting point for familiarizing yourself with E4X

Jon Udell: Introduction to E4X - Another tutorial/introduction. This article looks at some of the filtering and search capabilities of E4X not covered in the IBM article.

Adobe ActionScript 3 E4X API/overview - A full, readable reference to the E4X objects and their functionality.

Other Resources

For the sake of completeness, these are other sites I have found that discuss E4X, but are only worth reading if you are looking for more information

Wikipedia: E4X - This site provides an overview, description, and background of E4X, along with a few links, mostly listed here.

W3 Schools E4X Tutorial - This site also has a quick overview of E4X and a comparison to other approaches. It shows some sample code and how to manipulate XML objects, but does not go into detail.

javascript.faqts - Poorly organized, but packed with details of E4X and how to work with it. In FAQ format, it has questions on what methods are defined for XML and XMLList objects, and various other questions.

Standard ECMA-357 - The E4X Specification. This highly technical PDF is more geared toward implementing E4X than using it.
